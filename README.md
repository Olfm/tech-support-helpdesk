# Веб-сервис службы технической поддержки

Веб-приложение для приёма и обработки заявок в службу технической поддержки
учебного заведения. Пользователь подаёт заявку через сайт и выбирает её
категорию, а система передаёт заявку свободному оператору; заявитель видит
статус и переписку в личном кабинете. Помимо заявок есть раздел FAQ и форум.

Проект разработан как выпускная квалификационная работа на примере Департамента
электронного обучения и сопровождения ЧОУ ВО «Московский университет имени
С.Ю. Витте».

## Возможности

- Регистрация и вход пользователей, две роли: заявитель и оператор поддержки.
- Подача заявки с темой, описанием, категорией, приоритетом и контактами.
- Автоматическое распределение заявки на наименее загруженного оператора.
- Статусы заявки (новая, назначена, в работе, решена, закрыта) и переписка по ней.
- Уведомления о назначении, смене статуса и новых ответах.
- Отчёты для операторов: количество заявок по статусам и категориям, среднее
  время решения.
- Раздел FAQ с ответами на типовые вопросы.
- Форум с темами и ответами, закрытием тем оператором.
- Административная панель Django для управления пользователями, категориями,
  заявками, FAQ и форумом.

## Технологии

Python, Django 4.2 LTS, SQLite (по умолчанию) или MySQL, HTML, CSS. Шаблоны
Django, стандартная система аутентификации и групп прав.

## Структура проекта

```
config/      настройки, корневые маршруты, точки входа WSGI/ASGI
accounts/    регистрация, вход, роли (группа operator)
tickets/     заявки, категории, уведомления,
             распределение операторов (services.py), отчёты, сигналы
faq/         база знаний
forum/       темы и сообщения форума
templates/   HTML-шаблоны
static/      стили
```

Логика распределения заявок вынесена в `tickets/services.py`, поэтому её можно
проверять отдельно от веб-слоя (см. тесты).

## Быстрый старт

Нужен установленный Python 3.8 или новее.

```bash
# 1. Клонировать репозиторий и перейти в каталог
git clone https://github.com/Olfm/tech-support-helpdesk.git
cd tech-support-helpdesk

# 2. Создать и активировать виртуальное окружение
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Подготовить переменные окружения
cp .env.example .env

# 5. Применить миграции
python manage.py migrate

# 6. Загрузить демонстрационные данные (категории, пользователи, заявки)
python manage.py seed_demo

# 7. Запустить сервер разработки
python manage.py runserver
```

После запуска сайт доступен на http://127.0.0.1:8000/, админ-панель на
http://127.0.0.1:8000/admin/.

## Демонстрационные учётные записи

Команда `seed_demo` создаёт трёх пользователей. Если пароли не заданы через
переменные окружения, команда сгенерирует их и выведет в консоль один раз при
создании. Чтобы задать пароли заранее:

```bash
SEED_ADMIN_PASSWORD=... SEED_OPERATOR_PASSWORD=... SEED_USER_PASSWORD=... \
  python manage.py seed_demo
```

Учётные записи: `admin` (суперпользователь), `operator` (оператор поддержки),
`student` (обычный пользователь). Реальные пароли в репозитории не хранятся.

## Тесты

```bash
python manage.py test
```

Тесты покрывают выбор наименее загруженного оператора и автоматическое назначение и автораспределение, создание уведомлений при смене статуса,
разграничение доступа к чужим заявкам и отчётам, регистрацию и работу форума.

## Переключение на MySQL

По умолчанию используется SQLite, чтобы проект запускался без внешних сервисов.
Для MySQL в файле `.env` задайте `DB_ENGINE=mysql` и параметры подключения
(`DB_NAME`, `DB_USER`, `DB_PASSWORD`, `DB_HOST`, `DB_PORT`), затем установите
драйвер из `requirements.txt` (раскомментируйте `PyMySQL` или `mysqlclient`) и
выполните миграции.

## Конфигурация и безопасность

Настройки читаются из переменных окружения (файл `.env`), который не попадает в
репозиторий. Секретный ключ, режим отладки, список хостов и параметры БД не
зашиты в код. Для боевого развёртывания установите `DJANGO_DEBUG=False`, задайте
случайный `DJANGO_SECRET_KEY` и реальный список `DJANGO_ALLOWED_HOSTS`; в этом
режиме включаются защита cookies, HSTS и перенаправление на HTTPS.
